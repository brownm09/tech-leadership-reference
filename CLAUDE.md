# engineering-playbooks — Claude Code Configuration

## Repo Purpose

Reference playbooks for engineering leadership, incident management, and operational practice.
All content is Markdown; there is no compiled code.

## Testing

This is a documentation-only repository. There are no automated tests.

Before opening a PR, verify:
1. All new or modified Markdown files render correctly (check headers, links, footnotes)
2. New files are placed in the appropriate existing directory or a clearly named new one
3. Internal links (e.g., cross-references between playbooks) resolve to real paths
4. Footnote numbers are sequential and all `[^N]` markers have matching `[^N]:` definitions

Manual verification is sufficient — open the files in a Markdown preview (VS Code or GitHub PR
preview) and confirm formatting before requesting review.
